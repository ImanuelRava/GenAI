RAL-Bot Detailed Workflow

Redox-Active Ligands AI Assistant (Pure LLM - No RAG)

1
User Request & Validation
1.1
User Sends Query
POST request to /api/llm/chat endpoint for redox-active ligand questions
{ "message": "Explain redox-active ligands in catalysis", "provider": "gemini" }
1.2
Input Sanitization
• Sanitize input (max 4000 chars)
• Validate JSON structure
• No database lookup required
2
Direct LLM Processing
No RAG / No Database Context
2.1
System Prompt Definition
Specialized system prompt for redox-active ligand chemistry:
system_prompt = """You are RAL-Bot, an expert in redox-active ligands. Provide information about: - Ligand design principles - Electron transfer mechanisms - Catalytic applications of redox-active ligands"""
2.2
Provider Selection
Same provider selection as NiCOBot:
# Priority order: 1. Groq (GROQ_API_KEY) 2. Gemini (GEMINI_API_KEY) 3. HuggingFace (HF_API_KEY) 4. Ollama (local) 5. DeepSeek, OpenRouter, OpenAI, Anthropic
2.3
LLM API Call
Direct call without context enhancement:
llm = LLMProviderFactory.create(provider) response = llm.chat(system_prompt, user_message) # No retrieve_context() or build_enhanced_prompt()
3
Response Generation
3.1
Parse Response
Extract content from LLM response
return data['choices'][0]['message']['content']
3.2
Return to Client
JSON response:
{ "success": true, "response": "Redox-active ligands are molecules that...", "provider": "gemini" }
