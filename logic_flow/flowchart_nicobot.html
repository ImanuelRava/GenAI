NiCOBot Detailed Workflow

Nickel-catalyzed Cross-coupling Reactions AI Assistant with RAG

1
User Request & Validation
1.1
User Sends Query
POST request to /api/llm/chat with message, provider, api_key, model parameters
{ "message": "What is Suzuki coupling mechanism?", "provider": "groq" }
1.2
Input Validation
• Sanitize input (max 4000 chars)
• Validate JSON structure
• Check for empty message
message = sanitize_input(message, max_length=config.MAX_PROMPT_LENGTH)
↓
2
RAG Context Retrieval
2.1
Query Analysis
NiCOBotRAG.analyze_query() scores the query against keyword categories:
COMPOUND_KEYWORDS = ['smiles', 'electrophile', 'nucleophile', ...] PAPER_KEYWORDS = ['paper', 'doi', 'publication', ...] REACTION_KEYWORDS = ['suzuki', 'heck', 'kumada', ...]
2.2
Database Search
NiCOBotDatabase searches multiple indices:
Electrophiles
Nucleophiles
Papers
Reactions
2.3
Context Formatting
RAGContext object created with formatted context string for LLM prompt injection
context = RAGContext( compounds=[...], papers=[...], formatted_context="### Relevant Compounds:\n- ..." )
2.4
Prompt Enhancement
System prompt enhanced with retrieved database context
enhanced_prompt = f"{base_prompt}\n## Database Context\n{context.formatted_context}"
↓
