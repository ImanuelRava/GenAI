Deep Learning Fundamentals

Deep learning is a subset of machine learning based on artificial neural networks with multiple layers. These architectures have revolutionized AI, enabling breakthroughs in computer vision, natural language processing, and scientific discovery.

Why "Deep" Learning?

Deep networks have multiple hidden layers between input and output. Each layer learns increasingly abstract representations. Early layers might detect edges, middle layers detect shapes, and final layers recognize objects. This hierarchical learning is inspired by the visual cortex.

Neural Network Basics

Neural networks are composed of interconnected nodes (neurons) organized in layers. Each neuron performs a simple computation, but the collective behavior enables complex pattern recognition.

The Artificial Neuron

A single neuron computes a weighted sum of its inputs, adds a bias, and applies an activation function:

Single Neuron Computation:

$$z = \sum_i w_i x_i + b \quad \text{(Weighted sum + bias)}$$ $$a = f(z) \quad \text{(Activation function)}$$

Vectorized Form:

$$a = f(W \cdot x + b)$$

Activation Functions

Activation functions introduce non-linearity, enabling networks to learn complex patterns:

ReLU

$f(x) = \max(0, x)$. Most popular choice. Computationally efficient. Solves vanishing gradient for positive values.

Sigmoid

$\sigma(x) = \frac{1}{1+e^{-x}}$. Output in [0,1]. Used for binary classification output. Suffers from vanishing gradients.

Tanh

$\tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}$. Output in [-1,1]. Zero-centered. Still has vanishing gradient issue.

Softmax

$\text{softmax}(x_i) = \frac{e^{x_i}}{\sum_j e^{x_j}}$. Converts vector to probability distribution. Used for multi-class classification output. Sum outputs to 1.

Feedforward Networks

In a feedforward network, information flows in one direction from input to output through hidden layers:

1

Forward Pass

Input data propagates through each layer. Each layer: $z = W \cdot a_{\text{prev}} + b$, then $a = f(z)$. Final layer produces predictions.

2

Loss Computation

Compare predictions to true labels using a loss function (MSE for regression, cross-entropy for classification).

3

Backward Pass

Compute gradients of loss with respect to all weights and biases using backpropagation.

4

Weight Update

Adjust weights in the opposite direction of the gradient: $W_{\text{new}} = W_{\text{old}} - \alpha \cdot \frac{\partial L}{\partial W}$

Backpropagation & Optimization

Backpropagation is the algorithm for computing gradients efficiently in neural networks. Combined with gradient descent, it enables training of deep networks.

The Chain Rule

Backpropagation applies the chain rule recursively from output to input:

Chain Rule for Composition:

$$\frac{\partial L}{\partial w} = \frac{\partial L}{\partial a} \cdot \frac{\partial a}{\partial z} \cdot \frac{\partial z}{\partial w}$$

For layer $l$:

$$\delta^l = \frac{\partial L}{\partial z^l} = (W^{l+1})^T \cdot \delta^{l+1} \odot f'(z^l)$$

Weight gradients:

$$\frac{\partial L}{\partial W^l} = \delta^l \cdot (a^{l-1})^T$$ $$\frac{\partial L}{\partial b^l} = \delta^l$$

Optimization Algorithms

SGD

Stochastic Gradient Descent. Uses single samples or mini-batches. Simple but can be slow and stuck in local minima.

Momentum

Accumulates velocity in consistent gradient directions. Helps escape local minima and accelerate convergence.

Adam

Combines momentum with adaptive learning rates per parameter. Most popular optimizer. Good default choice.

Learning Rate Scheduling

Decrease learning rate over time. Warmup then decay. Helps fine-tune in later training stages.

The Vanishing/Exploding Gradient Problem

Why Deep Networks Were Hard to Train

In deep networks, gradients are multiplied many times during backpropagation. With activation functions like sigmoid (max derivative 0.25), gradients shrink exponentially—vanishing gradients. With large weights, they explode. Solutions: ReLU activation, batch normalization, residual connections, and careful initialization (Xavier/He).

Convolutional Neural Networks

CNNs are specialized for processing data with spatial structure, such as images. They use convolution operations to detect local patterns efficiently.

Key Components

Convolution Layer

Sliding filters (kernels) across input. Each filter detects a specific pattern (edge, texture). Parameters: kernel size, stride, padding.

Pooling Layer

Reduces spatial dimensions. Max pooling takes maximum in each window. Provides translation invariance and reduces computation.

Batch Normalization

Normalizes activations within a batch. Stabilizes training, allows higher learning rates, provides regularization effect.

Fully Connected

Final layers that combine features for classification. Often preceded by flattening or global average pooling.

Why CNNs Work

  • Local Connectivity: Each neuron connects only to a small region, reducing parameters dramatically
  • Parameter Sharing: Same filter applied across entire image—learns pattern detectors
  • Translation Equivariance: Patterns detected regardless of position
  • Hierarchical Features: Early layers: edges/textures; Middle: shapes/parts; Final: objects

Famous Architectures

Architecture Year Key Innovation
LeNet-5 1998 First successful CNN for digit recognition
AlexNet 2012 ReLU, Dropout, GPU training; started deep learning revolution
VGGNet 2014 Small 3×3 filters, very deep (16-19 layers)
ResNet 2015 Skip connections enabled 100+ layer networks
EfficientNet 2019 Compound scaling of depth, width, resolution

Recurrent Neural Networks

RNNs process sequential data by maintaining a hidden state that captures information from previous time steps. They're essential for text, time series, and biological sequences.

Basic RNN Architecture

RNN Forward Pass at time $t$:

$$h_t = f(W_{hh} \cdot h_{t-1} + W_{xh} \cdot x_t + b_h)$$ $$y_t = W_{hy} \cdot h_t + b_y$$

The hidden state $h_t$ carries information from past. Same weights ($W$) shared across all time steps.

The Vanishing Gradient Problem in RNNs

Standard RNNs struggle with long sequences because gradients vanish or explode over many time steps. The network "forgets" early inputs.

LSTM: Long Short-Term Memory

LSTMs solve the long-term dependency problem with gating mechanisms:

1

Forget Gate

Decides what information to discard from cell state. Output: $f_t = \sigma(W_f \cdot [h_{t-1}, x_t] + b_f)$

2

Input Gate

Decides what new information to store. Creates candidate values. $i_t = \sigma(W_i \cdot [h_{t-1}, x_t] + b_i)$

3

Cell State Update

Combine forget and input: $C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t$. The cell state is the "memory" that flows through.

4

Output Gate

Decides what to output based on cell state. $o_t = \sigma(W_o \cdot [h_{t-1}, x_t] + b_o)$; $h_t = o_t \odot \tanh(C_t)$

GRU: Gated Recurrent Unit

A simpler alternative to LSTM with fewer parameters. Combines forget and input gates into an "update gate." Often performs similarly to LSTM with faster training. Good choice when data is limited.

Transformers & Attention

Transformers have become the dominant architecture for NLP and are expanding to other domains. The key innovation is the self-attention mechanism that captures relationships regardless of distance.

Self-Attention Mechanism

Self-attention allows each position to attend to all other positions in the sequence:

Scaled Dot-Product Attention:

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

Where:
$Q$ (Query): What am I looking for?
$K$ (Key): What do I contain?
$V$ (Value): What do I provide?
$d_k$: Dimension of keys (scaling factor)

Multi-Head Attention:

$$\text{MultiHead}(Q,K,V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h)W^O$$ $$\text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V)$$

Key Components

Positional Encoding

Since attention has no notion of order, positional encodings are added to embeddings. Sinusoidal functions or learned embeddings.

Layer Normalization

Normalizes across features for each sample. Stabilizes training in deep transformers. Often applied before sublayers (Pre-LN).

Feed-Forward Network

Two linear layers with ReLU/GELU between. Applied to each position independently. Expands then contracts dimensions.

Residual Connections

Add input to output of each sublayer. Enables gradient flow in very deep networks. Essential for training transformers.

Encoder-Decoder Variants

  • BERT (Encoder-only): Bidirectional understanding, masked language modeling. Great for classification, NER, question answering.
  • GPT (Decoder-only): Autoregressive generation, causal attention. Powers ChatGPT, text generation, code completion.
  • T5/BART (Encoder-Decoder): Full sequence-to-sequence model. Good for translation, summarization, generation tasks.

The Scale Revolution

Transformer performance scales remarkably with model size and data. GPT-3 (175B parameters) showed emergent capabilities. Key insight: simple architecture + massive scale → powerful AI. This has driven the race to larger models.

Graph Neural Networks

GNNs operate on graph-structured data, making them ideal for molecular modeling, social networks, and biological systems. They learn representations by aggregating information from neighboring nodes.

Why Graphs for Chemistry?

Molecules are naturally represented as graphs where atoms are nodes and bonds are edges. GNNs can learn molecular representations directly from this structure without hand-crafted features.

Message Passing Framework:

For $k = 1$ to $K$ layers:

$$m_i^{(k)} = \sum_{j \in \mathcal{N}(i)} \text{MSG}(h_i^{(k-1)}, h_j^{(k-1)}, e_{ij})$$ $$h_i^{(k)} = \text{UPDATE}(h_i^{(k-1)}, m_i^{(k)})$$

Final Readout: $y = \text{READOUT}(\{h_i^{(K)}\})$ for graph-level prediction

Popular GNN Variants

GCN (Graph Convolutional)

Spectral-inspired approach. Averages neighbor features with normalized adjacency. Simple and effective for homogeneous graphs.

GAT (Graph Attention)

Uses attention to weight neighbors differently. Learns which neighbors are most relevant. Handles varying importance of neighbors.

Message Passing Neural Network

General framework for chemistry. Edge features (bond types) incorporated into messages. Basis for many molecular GNNs.

SchNet / DimeNet

3D-aware GNNs for molecules. Incorporates spatial information (distances, angles). Essential for property prediction.

Applications in Chemistry

  • Molecular Property Prediction: Predicting solubility, toxicity, binding affinity from structure
  • Reaction Prediction: Predicting products, reagents, and conditions
  • Molecule Generation: De novo drug design with desired properties
  • Retrosynthesis: Planning synthetic routes to target molecules
  • Protein-Ligand Binding: Predicting binding poses and affinities

Key Insight for Chemistry

GNNs respect molecular symmetry (permutation invariance). Rotating or renumbering atoms doesn't change the learned representation—critical for physical property prediction. Combined with proper training, GNNs can learn quantum mechanical properties from data.
