Supervised Learning

Supervised learning is the most common type of machine learning, where the algorithm learns from labeled training data to make predictions on new, unseen data. This module covers the fundamental algorithms, evaluation methods, and best practices for building effective supervised learning models.

Learning Objectives

  • Understand the difference between classification and regression problems
  • Master fundamental algorithms: Linear/Logistic Regression, Decision Trees, SVMs
  • Learn proper model evaluation metrics and validation techniques
  • Identify and address overfitting and underfitting issues
  • Apply supervised learning to chemistry and scientific problems

The Supervised Learning Paradigm

In supervised learning, we have input features X and corresponding target labels y. The goal is to learn a mapping function f: X → y that generalizes well to new data. The "supervision" comes from the labeled examples that guide the learning process.

Classification Algorithms

Classification is the task of predicting discrete labels or categories. It's one of the most widely used ML tasks, from spam detection to disease diagnosis.

Binary vs Multi-Class Classification

🔘

Binary Classification

Two possible outcomes (e.g., spam/not spam, toxic/non-toxic molecule). Output is typically a probability that's thresholded at 0.5.

🎯

Multi-Class Classification

Three or more possible categories (e.g., reaction type classification, compound activity level). Uses softmax for probability distribution.

Logistic Regression

Despite its name, logistic regression is used for classification. It models the probability of an instance belonging to a class using the sigmoid function.

Sigmoid Function:

$$\sigma(z) = \frac{1}{1 + e^{-z}}$$

Logistic Regression Model:

$$P(y=1|x) = \sigma(w \cdot x + b)$$

Cross-Entropy Loss:

$$L = -\sum \left[ y \cdot \log(\hat{y}) + (1-y) \cdot \log(1-\hat{y}) \right]$$

Key Properties

  • Probabilistic Output: Returns probabilities, not just class labels
  • Interpretable: Weights indicate feature importance and direction
  • Linear Decision Boundary: Can only separate linearly separable classes
  • Regularization: L1/L2 penalties prevent overfitting

Support Vector Machines (SVM)

SVMs find the optimal hyperplane that maximizes the margin between classes. They can handle non-linear boundaries using kernel functions.

Kernel Trick

Kernels transform data into higher-dimensional space where it becomes linearly separable. Common kernels: RBF (Gaussian), Polynomial, Sigmoid. The "kernel trick" computes these transformations efficiently without explicitly computing the high-dimensional coordinates.

Regression Algorithms

Regression predicts continuous numerical values. It's essential for tasks like predicting molecular properties, reaction yields, or binding affinities.

Linear Regression

The simplest regression model assumes a linear relationship between input features and the target variable.

Simple Linear Regression:

$$y = w \cdot x + b$$

Multiple Linear Regression:

$$y = w_1 x_1 + w_2 x_2 + \cdots + w_n x_n + b$$

Ordinary Least Squares (OLS):

$$\text{minimize} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2$$

Gradient Descent Update:

$$w_{\text{new}} = w_{\text{old}} - \alpha \cdot \frac{\partial L}{\partial w}$$

Assumptions of Linear Regression

  • Linearity: Relationship between X and y is linear
  • Independence: Observations are independent of each other
  • Homoscedasticity: Residuals have constant variance
  • Normality: Residuals are normally distributed

Regularized Regression

Ridge (L2)

Penalty: $\alpha \sum w^2$. Shrinks coefficients toward zero but never exactly zero. Good for multicollinearity.

Lasso (L1)

Penalty: $\alpha \sum |w|$. Can drive coefficients to exactly zero, performing feature selection automatically.

Elastic Net

Combines L1 and L2 penalties. Useful when features are correlated and you want feature selection.

Polynomial Regression

When relationships are non-linear, polynomial features can be added to capture curvature:

$$y = w_0 + w_1 x + w_2 x^2 + w_3 x^3 + \cdots + w_n x^n$$

Warning: Polynomial Degree

Higher polynomial degrees can lead to overfitting. The model may fit training data perfectly but generalize poorly. Always use cross-validation to select the optimal degree.

Decision Trees & Ensembles

Decision trees are intuitive models that split data based on feature values. Ensemble methods combine multiple trees for improved performance.

How Decision Trees Work

Decision trees recursively partition the feature space by selecting the best split at each node. The selection criteria differ for classification and regression:

Classification Trees

Gini Impurity: Measures the probability of misclassification. A pure node has Gini = 0.

$$\text{Gini} = 1 - \sum p_i^2$$

Regression Trees

Variance Reduction: Minimizes variance within each split. Predictions are mean values of leaf nodes.

$$\text{MSE} = \frac{\sum (y_i - \bar{y})^2}{n}$$

Ensemble Methods

1

Random Forest

Builds many decision trees on random subsets of data and features. Final prediction is the average (regression) or majority vote (classification). Reduces overfitting significantly.

2

Gradient Boosting

Builds trees sequentially, where each tree corrects errors of previous ones. Popular implementations: XGBoost, LightGBM, CatBoost. Often wins Kaggle competitions.

3

AdaBoost

Adaptive Boosting focuses on misclassified samples by adjusting their weights. Combines weak learners (often stumps) into a strong classifier.

Why Trees Work Well for Chemistry

Decision trees handle mixed data types (categorical and numerical), don't require feature scaling, and provide interpretable decision paths. They naturally handle missing values and are robust to outliers—making them ideal for molecular property prediction and QSAR modeling.

Evaluation Metrics

Choosing the right evaluation metric is crucial for model selection. Different metrics capture different aspects of model performance.

Classification Metrics

Metric Formula When to Use
Accuracy (TP + TN) / Total Balanced classes, equal cost of errors
Precision TP / (TP + FP) When false positives are costly
Recall TP / (TP + FN) When false negatives are costly
F1 Score 2 × (P × R) / (P + R) Balanced precision and recall
AUC-ROC Area under ROC curve Overall ranking performance

Regression Metrics

MAE (Mean Absolute Error)

Average of absolute errors. Robust to outliers. Same units as target variable.

MSE (Mean Squared Error)

Average of squared errors. Penalizes large errors more heavily. Differentiable for optimization.

RMSE

Square root of MSE. Same units as target. Most commonly reported metric.

R² (Coefficient of Determination)

Fraction of variance explained. 1.0 = perfect fit, 0.0 = baseline, <0 = worse than baseline.

Cross-Validation

Cross-validation provides a more robust estimate of model performance by using multiple train-test splits:

  • K-Fold CV: Split data into K folds, train on K-1, test on 1, repeat K times
  • Stratified K-Fold: Maintains class distribution in each fold (for classification)
  • Leave-One-Out: K = N (number of samples), computationally expensive but maximum data usage

Overfitting & Bias-Variance Tradeoff

Understanding the bias-variance tradeoff is essential for building models that generalize well to new data.

The Bias-Variance Decomposition

$$\text{Total Error} = \text{Bias}^2 + \text{Variance} + \text{Irreducible Error}$$

Where: Bias = systematic error (underfitting), Variance = sensitivity to training data (overfitting), Irreducible Error = noise in the data

Three Regimes

Underfitting (High Bias)

Model is too simple to capture patterns. High training error and high test error. Solution: Add features, increase model complexity, reduce regularization.

Optimal Fit

Balanced bias and variance. Training error slightly higher than perfect fit, but test error is minimized. This is the goal of model selection.

Overfitting (High Variance)

Model memorizes training data. Low training error, high test error. Solution: More data, regularization, simpler model, early stopping.

Regularization Techniques

1

L1/L2 Regularization

Add penalty terms to the loss function. L1 (Lasso) encourages sparsity, L2 (Ridge) shrinks weights smoothly.

2

Dropout (Neural Networks)

Randomly zero out neurons during training. Forces network to learn redundant representations.

3

Early Stopping

Stop training when validation error starts increasing. Prevents model from memorizing training data.

4

Data Augmentation

Create additional training examples through transformations. Particularly useful for image and molecular data.

Key Takeaway

The goal is not to minimize training error, but to minimize generalization error. Always evaluate on held-out test data, and use validation sets for hyperparameter tuning. A model that performs perfectly on training data but poorly on test data has failed its purpose.
